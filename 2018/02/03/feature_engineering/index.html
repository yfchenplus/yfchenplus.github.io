看Kaggle Criteo和Avazu的冠军如何做特征工程

发表于 2018-02-03 | 分类于 feature_engineering | 阅读次数:

本文主要讲在数据挖掘竞赛(Kaggle)中优秀的解决方案，看看大牛们是怎样做特征工程。比赛中大家用到的模型其实都差不多，比的就是特征工程。当然也有些人会想出一些新颖的模型。FFM模型就是作者参加Kaggle的Criteo的CTR预估比赛提出来的。通过阅读本文你会发现冠军获得者的特征工程做的都比较细致，非常巧妙。本文也会探讨一些小技巧背后的原因。

Kaggle Criteo 比赛

之前我有篇博客介绍了FFM模型，FFM模型的提出者就是本次比赛的冠军，之前模型的原理已经阐述过了，冠军的解决方案为 3 Idiots’ Approach for Display Advertising Challenge，个人觉得还是不够详细，下面详细介绍一些他们是怎么样做特征工程的。

比赛数据介绍

比赛的数据来自Criteo(美国的互联网广告公司)的历史七天点击和曝光数据。目标是预测接下来的一天用户点击广告的概率。数据格式如下表所示：

Label ${I}_{1}$ ${I}_{2}$ … ${I}_{13}$ ${C}_{1}$ ${C}_{2}$ … ${C}_{26}$
1 35 1 … 2741 68fd1e64 38a947a1 … null
0 7 91 … 1157 3516f6e6 cfc86806 … 796a1a2e

lable为1 (0)表示广告被点击 (未点击)。特征 ${I}_{1}$ 到 ${I}_{13}$ 为数值型特征，大多数为一些统计类的特征，特征${C}_{1}$到${C}_{26}$为离散类特征，这里的离散值已经被哈希了。

特征工程：

按上图的step分步介绍，并以如下样本举例：

symbol Label ${I}_{1}$ ${I}_{2}$ … ${I}_{13}$ ${C}_{1}$ ${C}_{2}$ … ${C}_{26}$
${x}_{raw}$ 1 35 1 … 2741 68fd1e64 38a947a1 … null

step 1

这一步是取出样本的数值型特征(Dense Feature),即特征${I}_{1}$到${I}_{13}$，对空值用-10填充，因为次数据集中的数值型特征都是大于等于0的，这样有区分度，数值型特征${x}_{dense}$格式如下表所示：

symbol Label ${I}_{1}$ ${I}_{2}$ … ${I}_{13}$
${x}_{dense}$ 1 35 1 … 2741

step 2

这一步是处理离散特征，首先是统计数据集中离散特征值的出现频率，下面是最高的26个离散特征取值。

${C}_{9}$=a73ee510, ${C}_{22}$=, ${C}_{17}$=e5ba7672, ${C}_{26}$=, ${C}_{23}$=32c7478e, ${C}_{6}$=7e0ccccf, ${C}_{14}$=b28479f6, ${C}_{19}$=21ddcdc9, ${C}_{14}$=07d13a8f, ${C}_{10}$=3b08e48b, ${C}_{6}$=fbad5c96, ${C}_{23}$=3a171ecb, ${C}_{20}$=b1252a9d, ${C}_{20}$=5840adea, ${C}_{6}$=fe6b92e5, ${C}_{20}$=a458ea53, ${C}_{14}$=1adce6ef, ${C}_{25}$=001f3601, ${C}_{22}$=ad3062eb, ${C}_{17}$=07c540c4, ${C}_{6}$=, ${C}_{23}$=423fab69, ${C}_{17}$=d4bb7bd8, ${C}_{2}$=38a947a1, ${C}_{25}$=e8b83407, ${C}_{9}$=7cc72ec2

基于上述列表我们可以把每一个样本映射成一个26维的向量，映射规则如下；如果样本的${C}_{9}$特征取值为a73ee510，那么第一维向量为1，否则为0。如果样本的${C}_{22}$特征取值为空，那么第二维向量为1，否则为0。以此类推，于是上面的${x}_{raw}$对应如下：

symbol Label ${C}_{9}$=a73ee510 ${C}_{22}$= … ${C}_{26}$= … ${C}_{2}$=38a947a1 ${C}_{25}$=e8b83407 ${C}_{9}$=7cc72ec2
${x}_{sparse}$ 1 0 0 … 1 … 1 0 0

step 3

把step 1生成的dense feature和step 2生成的 sparse feature拼接之后喂给GBDT（实现参考 ### Algorithm 5 ），GBDT模型参数为树高7，棵树30。GBDT的每棵树是一棵二叉树，对二叉树的从根节点依次编号1， 2，…。每个样本对应一个叶子节点编号(整数值)，对于每一个样本得到一个30维的特征向量。则${x}_{raw}$相对于GBDT生成的特征向量如下，假设${x}_{raw}$落在第一棵树编号为14的叶子节点， 落在第二棵树编号为15的叶子节点。

symbol Label ${Tree}_{1}$ ${Tree}_{2}$ … ${Tree}_{30}$
${x}_{GBDT}$ 1 14 15 … 23

step 4

数值型特征：如果该value大于2的话，则 value=int(math.log(float(value))**2)
离散特征：之前在step 2中统计过数据集中离散特征值的出现频率，如果离散特征值出现频率少于一定的阈值(为10),该离散特征取值为字符串 less。则${x}_{raw}$处理之后的特征为：

symbol Label ${I}_{1}$ ${I}_{2}$ … ${I}_{13}$ ${C}_{1}$ ${C}_{2}$ … ${C}_{26}$
${x}_{raw}$ 1 35 1 … 2741 68fd1e64 38a947a1 … null
${x}_{new}$ 1 12 1 … 62 less 38a947a1 … null

注：12 = int(math.log(float(35))**2)
${x}_{new}$和${x}_{new}$拼接

symbol Label ${I}_{1}$ ${I}_{2}$ … ${I}_{13}$ ${C}_{1}$ ${C}_{2}$ … ${C}_{26}$ ${Tree}_{1}$ … ${Tree}_{30}$
${x}_{new}+{x}_{GBDT}$ 1 12 1 … 62 less 38a947a1 … null 14 … 23

这里我们对域(field)进行编号，${I}_{1}$-${I}_{13}$对应${field}_{1}$-${field}_{13}$， ${C}_{1}$-${C}_{26}$对应${field}_{14}$-${field}_{39}$，${Tree}_{1}$-${Tree}_{30}$对应${field}_{40}$-${field}_{79}$，到后面我们会发现每一个域实际对应这6000个二值特征。上面的${x}_{new}+{x}_{GBDT}$在程序中的实际编码是如下格式的，即取值都是字符串形式：

${field}_{1}$ ${field}_{2}$ … ${field}_{13}$ ${field}_{14}$ ${field}_{15}$ … ${field}_{39}$ ${field}_{40}$ … ${field}_{79}$
${I}_{1}$ ${I}_{2}$ … ${I}_{13}$ ${C}_{1}$ ${C}_{2}$ … ${C}_{26}$ ${Tree}_{1}$ … ${Tree}_{30}$
‘I1-12’ ‘I2-SP1’ … ‘I13-62’ ‘C1-less’ ‘C13-8a947a1’ … ‘C26-‘ ‘1:14’ … ‘30:23’

关于此方案的在kaggle上作者的答疑

  1. 把低频特征转化成特殊的标志(即之前提到的less)。低频特征几乎包含很少信息，也很难让模型学到这些信息。实际上这些特征是有噪音的。通过这种操作，效果提升很明显(大于0.0005)

  2. 把数值型特征转化称离散特征。经验上使用离散特征效果上要好于数值特征。采用f(x)= int(math.log(float(x))**2), 是为了减少产生的离散特征的数量。该函数的离散效果还是蛮好的，函数的部分取值如下：

  3. 样本归一化。是为了模型更好的优化。举例来说 $x = (0, 0, 1, 1, 0, 1, 0, 1, 0)$，对$x$的每一个元素除以向量$x$的2范数，得到$ {x}_{normalized} = (0, 0, 0.5, 0.5, 0, 0.5, 0, 0.5, 0)$.

  4. hash技巧是为了特征表示能够简单处理，对效果并没有提升。并没有使用点击率特征
  5. 对随机梯度下降来说，学习率大会影响准确率，学习率小会收敛慢。

Kaggle Avazu

前面介绍了Criteo的比赛方案，接下来介绍Avazu的方案，冠军的解决方案为 4 Idiots’ Approach for Click-through Rate Prediction，。如下图所示，作者从原始数据集构造了30个子数据集，然后分别训练FFM模型，最后做集成。其实类似随机森林的思想，不过有三点不同：1.样本和特征不是随机采样，而是人工干预。2.这里的基分类器不是树，是FFM模型。3.输出也不是是简单的平均，具体是参考上面的pdf。这里只介绍其中的一个子数据集(即满足 ‘site_id’ = ‘85f751fd’所有样本),方案的数据流程图如下所示，图片来自这里：

比赛数据介绍

比赛的数据来自Criteo(美国的互联网广告公司)的历史11天点击和曝光数据。目标是预测用户点击广告的概率。这里是比赛官网。下面的是原始数据集的filed，所有的field都是离散型的，其中C1，C15，C16，C18，C19这五个field没有考虑, 其中括号的数字表示离散特征取值个数。

raw_fileds = [ id(40428967), click, hour(240), C1(7), banner_pos(7), site_id(4737), site_domain(7745), site_category(26), app_id(8552), app_domain(559), app_category(36), device_id(2686408), device_ip(6729486), device_model(8251), device_type(5), device_conn_type(4), C14(2616), C15(8), C16(9), C17(435), C18(4), C19(68), C20(172), C21(60) ]

我们之前考虑的样本满足’site_id’=’85f751fd’，个人臆测这表示这条广告来自app应用，不是site，所以 site_id, site_domain, site_category这三个field也没有用。于是得到FILEDS如下所示：

FIELDS = [ id, click, hour, banner_pos, app_id, app_domain, app_category, device_id, device_ip, device_model, device_type, device_conn_type, C14, C17, C20, C21 ]

下面构造了一些统计类的特征，不考虑是否点击，具体说明如下表所示，其中threshold与下面的代码对应，范围表示字段的取值范围，percent解释如下，数据集中有18%的样本满足字段device_id_count>1500。

new field comment threshold 范围 percent
device_id_count device_id 的出现次数 1000 (0， 228637) 18%
device_ip_count device_ip 的出现次数 1000 (0，37300009) 93%

关于此方案的在kaggle上作者的答疑

  1. 为什么不用离散特征C14,C17,C20,C21
    特征选择是在采样后的数据集上进行的（整个数据集的10%），使用所有的特征作为一个baseline。如果在删除一个特征后，效果没有变化或者效果提升，则删掉该特征。

  2. 如何选择不同的sbset
    如果一个子集在加入ensemble之后效果有提升，则添加该子集。

  3. 和criteo的比赛相比，为什么不用GBDT来生成特征
    主要原因是数据集里没有数值型的特征，基于树的模型比较适合数值型特征。
